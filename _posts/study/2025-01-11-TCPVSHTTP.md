# TCP?

## TCP란 무엇인가?

TCP(Transmission Control Protocol)는 인터넷에서 데이터 전송할 때 쓰는 **연결 지향 프로토콜**임. 데이터가 잘 도착하게 하고 순서 보장함. 오류나 손실 발생 시 복구도 알아서 함. 주요 특징:

- **신뢰성**: 데이터가 순서대로 도착하고, 손실 없게 보장.
- **흐름 제어**: 송/수신 속도 맞춰서 네트워크 혼잡 방지.
- **오류 제어**: 오류 생기면 재전송해서 복구.
- **세그먼트화**: 큰 데이터 나눠서 보내고, 다시 조립함.
- **3-웨이 핸드셰이크**: 연결 설정하려고 세 번 신호 주고받음.

이메일, 파일 전송, 웹 브라우징 등에서 다 씀.

# **TCP와 HTTP의 차이점**

## 계층 위치와 역할

- **TCP**: 전송 계층 → 데이터 전달 및 신뢰성 보장.
- **HTTP**: 애플리케이션 계층 → 데이터를 사용하는 규칙과 형식 정의.

## 데이터 처리 방식

- **TCP**: 데이터를 **패킷 단위**로 나눠 전송, 순서 보장, 오류 복구 처리.
- **HTTP**: **요청/응답 구조**로 동작. 클라이언트가 요청하면 서버가 응답하는 방식.

## 연결 관리

- **TCP**: 연결 기반. 3-웨이 핸드셰이크로 연결 설정, 흐름/혼잡 제어 가능.
- **HTTP**: 기본적으로 **비연결성** 기반. 요청 후 응답 끝나면 연결 종료.
  - 단, HTTP/1.1에서는 **Keep-Alive** 옵션으로 연결 유지 가능.
  - HTTP/2부터 멀티플렉싱 지원으로 성능 개선.

# **TCP와 HTTP의 관계**

## HTTP가 TCP 위에서 동작하는 원리

HTTP는 TCP를 기반으로 작동하는 **애플리케이션 계층 프로토콜**임.

동작 원리:

1. **TCP 연결 설정**: HTTP 요청을 보내기 전, 클라이언트와 서버는 3-웨이 핸드셰이크를 통해 TCP 연결을 설정함.
2. **HTTP 데이터 전송**: 클라이언트가 HTTP 요청(예: GET, POST)을 TCP 연결을 통해 전송하고, 서버는 요청에 대한 응답을 TCP를 통해 전달함.
3. **TCP의 신뢰성 사용**: HTTP가 전송하는 데이터는 TCP의 패킷 단위로 나눠져 전송됨. TCP는 데이터 순서 보장, 손실 복구, 흐름 제어 등의 역할을 맡음.
4. **연결 종료**: 요청-응답이 끝나면 TCP 연결 종료. (HTTP/1.1의 Keep-Alive로 연결 유지 가능)

## HTTP/2와 HTTP/3의 변화

### **HTTP/2: TCP 기반 멀티플렉싱**

- **멀티플렉싱 지원**: 하나의 TCP 연결에서 여러 요청과 응답을 동시에 처리.→ 이전의 HTTP/1.1에서 발생하던 **헤드 오브 라인 블로킹(HOL Blocking)** 일부 해결.
- **헤더 압축**: 헤더 크기 줄여서 데이터 전송 효율 향상.
- TCP의 신뢰성을 그대로 사용하지만, TCP의 근본적인 한계(네트워크 손실 시 전체 HOL Blocking 발생)로 인해 성능 제한 존재.

---

### **HTTP/3: QUIC 프로토콜로 전환**

- **TCP 대신 QUIC 사용**: UDP 기반의 QUIC 프로토콜로 전환해 성능 개선.
  - 연결 설정 속도 향상: 3-웨이 핸드셰이크 대신 초기 핸드셰이크에서 보안(TLS)과 연결 설정 동시 처리.
  - 패킷 손실에 강함: 독립적인 스트림 전송으로 **HOL Blocking 완전 해결**.
- **내장 보안**: QUIC는 기본적으로 TLS 암호화 통합.
- 결과: 더 빠른 연결, 낮은 지연 시간, 네트워크 환경에서의 강건성 강화.

# **결론**

## TCP와 HTTP의 협력으로 이루어진 인터넷 통신

- TCP와 HTTP는 인터넷 통신에서 각각 다른 역할을 수행하면서 상호 보완적으로 작동함.
  - **TCP**: 데이터를 신뢰성 있게 전달하는 기반 기술로, 패킷 단위 전송, 오류 복구, 순서 보장을 담당.
  - **HTTP**: 데이터를 활용하는 방식과 구조를 정의하며, 주로 클라이언트와 서버 간 요청-응답을 처리.
- **HTTP는 TCP 위에서 동작**하며, TCP의 신뢰성을 활용해 데이터를 안전하게 교환함.
  - TCP가 없으면 HTTP가 제공하는 웹 데이터의 신뢰성 보장이 어려움.
  - HTTP의 성능 향상은 TCP의 구조와 한계를 고려한 최적화 작업과 직결됨.
- **HTTP/2와 HTTP/3의 발전**은 TCP와의 관계를 재정립하거나 새로운 전송 기술(QUIC)을 도입하며, 인터넷 성능을 혁신적으로 개선.
- 결과적으로, TCP와 HTTP는 인터넷의 핵심 통신 프로토콜로, 서로의 강점을 극대화하여 현대 인터넷 환경을 가능하게 함.

인터넷은 **TCP의 신뢰성**과 **HTTP의 유연성**이라는 협력을 기반으로 발전해 왔고, 앞으로도 이 관계를 바탕으로 더욱 효율적이고 빠른 통신 기술이 개발될 것임.
